📋 Tổng quan hệ thống

🤖 AI Pipeline

Sử dụng Ollama local để phân tích nhanh, Gemini để chuẩn hoá đặc tả/golden oracle, sau đó backend tự sinh validator và generator theo input_schema.

🛡️ Sandbox Bảo mật

Code được thực thi trong sandbox cô lập với giới hạn bộ nhớ, CPU và tiến trình. Chống fork bomb, OOM.

🗄️ Database HA

MySQL Master/Slave được cấu hình trong Docker Compose. App dùng master cho ghi, slave cho các luồng đọc khi cấu hình datasource tương ứng.

⚡ Xử lý bất đồng bộ

Các tác vụ nặng (sinh test case, chấm bài) chạy trên thread pool riêng. Frontend polling để cập nhật kết quả.

🔑 Lấy API Keys

1

Gemini API Key (Google AI Studio)

Dùng cho OCR ảnh đề, chuẩn hoá metadata, sinh golden solution nội bộ và kế hoạch kiểm thử.

  1. Truy cập aistudio.google.com/app/apikey
  2. Đăng nhập bằng tài khoản Google
  3. Nhấn "Create API key"
  4. Copy key và lưu vào file .env
2

Ollama Local

Dùng để phân tích nhanh đề bài trên máy local trước khi Gemini sinh artifact kiểm thử.

  1. Cài Ollama từ ollama.com
  2. Chạy ollama pull llama3.1:8b
  3. Đảm bảo Ollama đang lắng nghe tại http://localhost:11434
  4. Cấu hình model trong file .env nếu muốn đổi model.
3

Tạo file .env tại thư mục gốc dự án

GEMINI_API_KEY=AIzaSy...your_key_here
# Hoặc dùng nhiều key, phân tách bằng dấu phẩy/chấm phẩy/xuống dòng
GEMINI_API_KEYS=
GEMINI_MODEL=gemini-2.5-flash
GEMINI_PRO_MODEL=gemini-2.5-flash

# Khi chạy bằng Docker, app gọi Ollama trên máy host qua host.docker.internal
OLLAMA_BASE_URL=http://host.docker.internal:11434
OLLAMA_MODEL=llama3.1:8b
🔒
File .env đã được thêm vào .gitignore. Không commit file này lên Git.

📦 Cài đặt dự án

1

Clone repository

git clone https://github.com/your-username/pbj-judge.git
cd pbj-judge
2

Tạo file .env với API keys (xem mục trên)

GEMINI_API_KEY=your_gemini_api_key
GEMINI_API_KEYS=
GEMINI_MODEL=gemini-2.5-flash
GEMINI_PRO_MODEL=gemini-2.5-flash
OLLAMA_BASE_URL=http://host.docker.internal:11434
OLLAMA_MODEL=llama3.1:8b
3

Build mã nguồn Java bằng Maven

mvn clean package -DskipTests
ℹ️
Lần đầu chạy sẽ tốn thời gian tải dependencies (~2-3 phút). Các lần sau sẽ nhanh hơn do được cache.
4

Khởi động toàn bộ hệ thống bằng Docker Compose

docker compose up -d --build

Docker sẽ tự động:
✅ Build image ứng dụng từ Dockerfile
✅ Khởi động MySQL Master (port 3307) và MySQL Slave (port 3308)
✅ Đợi database sẵn sàng (healthcheck) rồi mới chạy app
✅ Gắn volume testcase-data để lưu file test case trong container

5

Xác nhận hệ thống đang chạy

docker ps

Bạn sẽ thấy 3 container đang chạy:

ContainerPortMô tả
pbj-judge-app8080Ứng dụng Spring Boot chính
mysql-master3307MySQL Master (đọc + ghi)
mysql-slave3308MySQL Slave (chỉ đọc)

Luồng sử dụng cơ bản

1

Tạo bài tập mới

Nhấn "Tạo bài mới" → Nhập tiêu đề → Upload ảnh đề bài (tùy chọn) hoặc nhập mô tả → Nhấn "Phân Tích & Tạo Đề". AI sẽ xử lý trong nền (30–90 giây).

2

Kiểm tra bài tập

Nhấn vào tên bài tập → Xem đề bài, test case mẫu → Nhập code → Chọn ngôn ngữ → Nhấn "Chạy mã nguồn & Kiểm tra".

3

Tạo lại test case

Trong trang chi tiết bài tập → Nhấn nút "Tạo lại" → Xác nhận → Đợi AI sinh test case mới.

Các lệnh quản trị hữu ích

# Xem log ứng dụng
docker logs pbj-judge-app --tail 50 -f

# Khởi động lại app (không rebuild)
docker compose restart app

# Dừng toàn bộ hệ thống
docker compose down

# Dừng và XÓA dữ liệu database (cẩn thận!)
docker compose down -v

# Rebuild và khởi động lại sau khi sửa code
mvn clean package -DskipTests && docker compose up -d --build app

Kiến trúc AI Pipeline

Bước 1 Người dùng nhập đề bài (text + ảnh)
Bước 2 Nếu có ảnh, Gemini OCR trích xuất statement; ảnh được encode ngay trước khi job chạy nền
Bước 3 Ollama local phân tích → sinh analysis_json ngắn
Bước 4 Gemini nhận đề gốc + analysis_json → sinh metadata, input_schema, golden oracle, test plan
Bước 5 Backend tự dựng validator từ input_schema và chọn generator hệ thống/fallback
Bước 6 Chạy generator + validator + golden solution trong sandbox, có kiểm tra multi-testcase và profile coverage
Bước 7 Lưu .in/.out và cache artifact đã qua validation

🛠️ Xử lý lỗi thường gặp

!

Container không khởi động được

docker logs pbj-judge-app 2>&1 | tail -30

Nếu lỗi "Connection refused" → MySQL chưa sẵn sàng. Đợi thêm 30 giây rồi chạy lại docker compose up -d.

!

Lỗi 401 Unauthorized từ AI

API key không hợp lệ, hết quota, hoặc Ollama chưa chạy. Kiểm tra GEMINI_API_KEY/GEMINI_API_KEYS, OLLAMA_BASE_URL và service Ollama local.

!

Lỗi BUILD FAILURE khi chạy Maven

# Kiểm tra phiên bản Java
java -version   # Phải là 21

# Xóa cache và build lại
mvn clean package -DskipTests -U
!

Lỗi formal spec báo unknown/unspecified

Thường do OCR thiếu phần Input/Output/Constraints hoặc mô tả text quá ít. Hãy nhập thêm statement dạng text hoặc upload ảnh rõ hơn. Log cần xem: docker logs pbj-judge-app --tail 200.

!

Lỗi golden solution fail/TLE khi sinh testcase

Backend sẽ log lỗi compile/runtime/TLE chi tiết và tự xoá cache artifact hỏng khi cần. Với bài nhiều test, validator/generator hiện đã hỗ trợ multiple_test_cases=true.

!

Test case không hiển thị sau khi tạo bài

Kiểm tra thư mục testcase_data/ — nếu trống thì generator đã thất bại. Xem log: docker logs pbj-judge-app | grep "DEBUG\|ERROR". Thử nhấn "Tạo lại" trong trang bài tập.

!

Port 8080 đã bị chiếm

# Tìm process đang dùng port 8080
netstat -ano | findstr :8080

# Hoặc đổi port trong docker-compose.yml
ports:
  - "8081:8080"   # Dùng port 8081 thay thế
