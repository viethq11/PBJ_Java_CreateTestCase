Tổng quan hệ thống

Luồng tạo bài + test case

Người dùng nhập tiêu đề, mô tả hoặc ảnh đề bài. Backend tạo job nền, dùng AI để chuẩn hoá đề, dựng schema, sinh test plan, sinh input/output và lưu test case.

AI hai tầng

Ollama local phân tích nhanh nội dung. Gemini xử lý OCR, metadata, input schema, golden oracle và kế hoạch kiểm thử có kiểm tra semantic.

Judge sandbox

Hệ thống chạy C++, Java và Python với giới hạn thời gian/bộ nhớ. Kết quả hỗ trợ AC, WA, TLE, CE, RE và so sánh với trạng thái kỳ vọng.

Lưu trữ test case

Test case được lưu qua TestCaseStorageService. Khi chạy Docker, dữ liệu nằm trong volume testcase-data được mount vào /app/testcase-data.

API keys và biến môi trường

1

Lấy Gemini API key

  1. Vào Google AI Studio - API keys.
  2. Đăng nhập tài khoản Google.
  3. Chọn Create API key, copy key và lưu lại.
  4. Nếu có nhiều key, đặt vào GEMINI_API_KEYS để hệ thống có thể xoay vòng khi hết quota.
2

Cài và chuẩn bị Ollama

Project này yêu cầu cài model Ollama deepseek-coder:6.7b. Chỉ cài Ollama mà chưa pull model thì bước phân tích local sẽ thất bại.

ollama pull deepseek-coder:6.7b
ollama list
ollama serve

Trên nhiều máy, Ollama tự chạy nền sau khi cài. Kiểm tra bằng cách mở http://localhost:11434 hoặc chạy ollama list.

3

Tạo file .env ở thư mục gốc

GEMINI_API_KEY=your_gemini_api_key
GEMINI_API_KEYS=
GEMINI_MODEL=gemini-2.5-flash
GEMINI_PRO_MODEL=gemini-2.5-flash
GEMINI_TIMEOUT_SECONDS=240

# Docker app gọi Ollama trên máy host qua host.docker.internal.
OLLAMA_BASE_URL=http://host.docker.internal:11434
OLLAMA_MODEL=deepseek-coder:6.7b

# Giữ 1 để tránh nhiều job AI chạy song song làm cạn quota/RAM.
AI_QUEUE_MAX_CONCURRENCY=1
✓
File .env đã nằm trong .gitignore. Không commit API key lên Git.

  • Spring Boot Extension Pack để chạy app và xem bean/config.
  • Docker để xem container, log và compose service.

2

Mở và chạy

  1. Chọn File → Open Folder → mở thư mục PBJ_JUDGE.
  2. Chờ VS Code import Maven project.
  3. Chạy Docker bằng terminal: docker compose up -d --build.
  4. Hoặc chạy local: mở class com.pbj.PbjApplication và bấm Run sau khi database đã chạy.

1

  4. Nếu dùng bản Community, chạy bằng Maven hoặc class main. Nếu dùng Ultimate, có thể dùng Spring Boot run configuration.

Hướng dẫn sử dụng hệ thống sinh test case

Luồng trên giao diện web

1

Mở trang tạo bài

Truy cập http://localhost:8080, bấm Tạo bài mới. Trang tạo bài nằm tại /create.

2

Nhập dữ liệu đề bài

  • Tiêu đề bài toán: bắt buộc, ví dụ Dãy con tăng dài nhất.
  • Ảnh minh hoạ: tuỳ chọn, có thể upload nhiều ảnh. Gemini sẽ OCR nội dung ảnh.
  • Mô tả yêu cầu: nên có statement, input, output, constraints và sample để AI ít phải suy đoán.
i
Đề càng rõ phần Input/Output/Constraints thì validator và generator càng ổn định. Nếu chỉ upload ảnh mờ hoặc thiếu constraints, hệ thống có thể trả trạng thái cần bổ sung thông tin.
3

Bấm Phân Tích & Tạo Đề

Frontend gửi form tới POST /api/problem/generate. Backend trả jobId, sau đó giao diện tự polling GET /api/job/{id} mỗi 2.5 giây.

4

Chờ job hoàn tất

Trạng tháiÝ nghĩaViệc cần làm

Test case được sinh như thế nào?

1Nhận đề bài dạng text và ảnh.

2Nếu có ảnh, Gemini OCR để lấy statement.

3Ollama phân tích nhanh loại bài, ràng buộc và cấu trúc input.

4Gemini chuẩn hoá metadata, sinh input_schema, test_plan và golden oracle.

5Backend dựng validator local từ schema, chọn generator chuyên biệt hoặc fallback generator.

6Hệ thống sinh nhiều profile test: sample, basic, edge, boundary, random, stress/adversarial nếu phù hợp.

7Golden solution chạy trong sandbox để tạo output; validator kiểm tra input/output trước khi lưu.

Xem và dùng test case

1

Trang chi tiết bài tập

Tại /problem/{id}, bạn sẽ thấy statement, input format, output format, constraints, kế hoạch kiểm thử AI và danh sách test case đã tạo. Test lớn có thể chỉ hiển thị bản xem trước để tải trang nhanh hơn, nhưng judge vẫn dùng file đầy đủ.

2

Chạy code để kiểm tra

  1. Chọn ngôn ngữ: C++, Java hoặc Python.
  2. Chọn kết quả kỳ vọng: AC, WA hoặc TLE.
  3. Dán source code vào editor bên phải.
  4. Bấm Chạy mã nguồn & Kiểm tra.

Frontend gửi tới POST /api/problem/run và polling job giống luồng tạo bài.

Tạo lại test case

1

Khi nào nên tạo lại?

  • Test case quá ít hoặc chưa bao phủ edge case.
  • Bạn vừa sửa mô tả đề bài, constraints hoặc input format.
  • Golden output có dấu hiệu sai hoặc generator sinh dữ liệu không hợp lệ.
2

Cách tạo lại

Trong trang chi tiết bài, bấm Tạo lại, xác nhận cảnh báo. Giao diện gọi POST /api/problem/regenerate-testcases. Hành động này xoá test case hiện tại của bài và sinh lại bộ mới.

Mẹo viết mô tả để sinh test tốt

Tên bài: Tổng đoạn con lớn nhất

Mô tả:
Cho mảng A gồm n số nguyên. Tìm tổng lớn nhất của một đoạn con liên tiếp không rỗng.

Input:
Dòng 1 chứa số nguyên n.
Dòng 2 chứa n số nguyên A_i.

Output:
In ra tổng lớn nhất của một đoạn con liên tiếp.

Constraints:
1 <= n <= 200000
-10^9 <= A_i <= 10^9

Sample:
Input
5
-2 3 -1 4 -5
Output
6

Xử lý lỗi thường gặp

Không truy cập được http://localhost:8080

docker ps
docker logs pbj-judge-app --tail 100

Nếu port bị chiếm, đổi mapping trong docker-compose.yml từ 8080:8080 sang 8081:8080, rồi truy cập http://localhost:8081.

!

Lỗi kết nối MySQL

Chờ healthcheck hoàn tất rồi chạy lại app. Khi chạy local từ IDE, nhớ dùng port host 3307/3308, không phải port container 3306.

docker compose logs -f mysql-master
docker compose restart app
!

Gemini báo 401, 403 hoặc hết quota

  • Kiểm tra GEMINI_API_KEY trong .env.
  • Nếu dùng nhiều key, điền GEMINI_API_KEYS.
  • Restart app sau khi sửa .env: docker compose up -d --build app.
!

Ollama không phản hồi

ollama list
ollama pull deepseek-coder:6.7b
ollama serve

Nếu chạy app trong Docker, dùng OLLAMA_BASE_URL=http://host.docker.internal:11434. Nếu chạy app local ngoài Docker, dùng http://localhost:11434.

!

Job sinh test case trả NEEDS_INPUT

Hệ thống chưa đủ thông tin để dựng schema an toàn. Bổ sung rõ các mục Input, Output, Constraints, sample và quy tắc đặc biệt như nhiều test case, modulo, định dạng dòng.

!

Golden solution fail, TLE hoặc output sai

Xem log AI/debug để biết lỗi compile/runtime/TLE. Thường nguyên nhân là đề thiếu ràng buộc, input format mơ hồ hoặc bài cần thuật toán đặc biệt chưa được mô tả đủ.

docker logs pbj-judge-app --tail 300 | findstr /i "ERROR DEBUG golden validator generator"
!

Maven build failure

java -version
mvn -version
mvn clean package -DskipTests -U

Đảm bảo Java đang là JDK 21. Nếu IDE vẫn báo lỗi Lombok, bật annotation processing hoặc cài plugin Lombok.
