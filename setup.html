📋 Tổng quan hệ thống

🤖 AI Pipeline

Sử dụng Gemini 2.0 Flash để phân tích đề bài và Groq (Llama 3.3 70B) để sinh test case tốc độ cao.

🛡️ Sandbox Bảo mật

Code được thực thi trong sandbox cô lập với giới hạn bộ nhớ, CPU và tiến trình. Chống fork bomb, OOM.

🗄️ Database HA

MySQL Master-Slave replication: Master xử lý ghi, Slave xử lý đọc — tăng hiệu năng và độ tin cậy.

⚡ Xử lý bất đồng bộ

Các tác vụ nặng (sinh test case, chấm bài) chạy trên thread pool riêng. Frontend polling để cập nhật kết quả.

🔑 Lấy API Keys

1

Gemini API Key (Google AI Studio)

Dùng để phân tích đề bài và sinh code mẫu (AC/WA/TLE).

  1. Truy cập aistudio.google.com/app/apikey
  2. Đăng nhập bằng tài khoản Google
  3. Nhấn "Create API key"
  4. Copy key và lưu vào file .env
2

Groq API Key (Groq Cloud)

Dùng để sinh test case tốc độ cao qua mô hình Llama 3.3 70B.

  1. Truy cập console.groq.com/keys
  2. Đăng ký / Đăng nhập tài khoản
  3. Nhấn "Create API Key"
  4. Copy key và lưu vào file .env
3

Tạo file .env tại thư mục gốc dự án

GEMINI_API_KEY=AIzaSy...your_key_here
GROQ_API_KEY=gsk_...your_key_here
🔒
File .env đã được thêm vào .gitignore. Không commit file này lên Git.

📦 Cài đặt dự án

1

Clone repository

git clone https://github.com/your-username/pbj-judge.git
cd pbj-judge
2

Tạo file .env với API keys (xem mục trên)

GEMINI_API_KEY=your_gemini_api_key
GROQ_API_KEY=your_groq_api_key
3

Build mã nguồn Java bằng Maven

mvn clean package -DskipTests
ℹ️
Lần đầu chạy sẽ tốn thời gian tải dependencies (~2-3 phút). Các lần sau sẽ nhanh hơn do được cache.
4

Khởi động toàn bộ hệ thống bằng Docker Compose

docker-compose up -d --build

Docker sẽ tự động:
✅ Build image ứng dụng từ Dockerfile
✅ Khởi động MySQL Master (port 3307) và MySQL Slave (port 3308)
✅ Đợi database sẵn sàng (healthcheck) rồi mới chạy app
✅ Tạo thư mục testcase_data/ để lưu trữ dữ liệu test case

5

Xác nhận hệ thống đang chạy

docker ps

Bạn sẽ thấy 3 container đang chạy:

ContainerPortMô tả
pbj-judge-app8080Ứng dụng Spring Boot chính
mysql-master3307MySQL Master (đọc + ghi)
mysql-slave3308MySQL Slave (chỉ đọc)

Luồng sử dụng cơ bản

1

Tạo bài tập mới

Nhấn "Tạo bài mới" → Nhập tiêu đề → Upload ảnh đề bài (tùy chọn) hoặc nhập mô tả → Nhấn "Phân Tích & Tạo Đề". AI sẽ xử lý trong nền (30–90 giây).

2

Kiểm tra bài tập

Nhấn vào tên bài tập → Xem đề bài, test case mẫu → Nhập code → Chọn ngôn ngữ → Nhấn "Chạy mã nguồn & Kiểm tra".

3

Tạo lại test case

Trong trang chi tiết bài tập → Nhấn nút "Tạo lại" → Xác nhận → Đợi AI sinh test case mới.

Các lệnh quản trị hữu ích

# Xem log ứng dụng
docker logs pbj-judge-app --tail 50 -f

# Khởi động lại app (không rebuild)
docker-compose restart app

# Dừng toàn bộ hệ thống
docker-compose down

# Dừng và XÓA dữ liệu database (cẩn thận!)
docker-compose down -v

# Rebuild và khởi động lại sau khi sửa code
mvn clean package -DskipTests && docker-compose up -d --build

Kiến trúc AI Pipeline

Bước 1 Người dùng nhập đề bài (text + ảnh)
Bước 2 Gemini 2.5 Flash phân tích → sinh JSON (mô tả, format I/O, constraints, generator code, golden solution)
Bước 3 Generator script chạy trong sandbox → sinh input data với nhiều seed
Bước 4 Golden solution (C++) chạy trên mỗi input → sinh expected output
Bước 5 File .in/.out lưu vào testcase_data/problem_X/
Bước 6 Kiểm tra verdict separation: AC solution pass, WA probe fail, TLE probe timeout

🛠️ Xử lý lỗi thường gặp

!

Container không khởi động được

docker logs pbj-judge-app 2>&1 | tail -30

Nếu lỗi "Connection refused" → MySQL chưa sẵn sàng. Đợi thêm 30 giây rồi chạy lại docker-compose up -d.

!

Lỗi 401 Unauthorized từ AI

API key không hợp lệ hoặc hết quota. Kiểm tra file .env và thử key mới tại Google AI Studio / Groq Console.

!

Lỗi BUILD FAILURE khi chạy Maven

# Kiểm tra phiên bản Java
java -version   # Phải là 21

# Xóa cache và build lại
mvn clean package -DskipTests -U
!

Test case không hiển thị sau khi tạo bài

Kiểm tra thư mục testcase_data/ — nếu trống thì generator đã thất bại. Xem log: docker logs pbj-judge-app | grep "DEBUG\|ERROR". Thử nhấn "Tạo lại" trong trang bài tập.

!

Port 8080 đã bị chiếm

# Tìm process đang dùng port 8080
netstat -ano | findstr :8080

# Hoặc đổi port trong docker-compose.yml
ports:
  - "8081:8080"   # Dùng port 8081 thay thế
💡
Tip: Để xem log realtime của cả hệ thống: docker-compose logs -f
